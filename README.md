## Быстрый старт

**Требования:** Python 3.9+, pip.

```bash
# 1. Клонировать репозиторий
git clone <url-репозитория>
cd test-proj-fastapi

# 2. Создать и активировать виртуальное окружение
python -m venv .venv
source .venv/bin/activate  # macOS / Linux

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Настроить переменные окружения
cp .env.example .env
# Отредактировать .env — обязательно указать OPENROUTER_API_KEY
# (получить ключ можно на https://openrouter.ai/keys)

# 5. Запустить сервер
uvicorn app.main:app --reload
```

После запуска:

- Фронтенд: <http://127.0.0.1:8000/>
- Swagger-документация: <http://127.0.0.1:8000/docs>
- ReDoc: <http://127.0.0.1:8000/redoc>
- OpenAPI JSON: <http://127.0.0.1:8000/openapi.json>

### Переменные окружения

Файл `.env`:

```ini
# AI (OpenRouter) — обязательное поле
OPENROUTER_API_KEY=sk-or-v1-...

# Опционально: модель (по умолчанию openrouter/free)
OPENROUTER_MODEL=openrouter/free

# SMTP для email-уведомлений (необязательно)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your@email.com
SMTP_PASSWORD=app-password
SMTP_FROM=noreply@portfolio.dev
SMTP_TO=owner@email.com

# Rate Limiting (по умолчанию 5 запросов в час)
RATE_LIMIT_MAX_REQUESTS=5
RATE_LIMIT_WINDOW_SECONDS=3600
```

**Важно:** Для работы аналитики AI обязателен `OPENROUTER_API_KEY`. Без него сервис будет возвращать fallback-значения (`"неопределен"` / `"общее"`). Email-уведомления опциональны — при отсутствии SMTP-настроек письма имитируются.

## Стек технологий

| Компонент | Технология |
|-----------|-----------|
| **Язык** | Python 3.14 (совместимость с 3.9+) |
| **Фреймворк** | FastAPI 0.115+ |
| **ASGI-сервер** | Uvicorn 0.32+ |
| **Валидация / сериализация** | Pydantic 2.9+ (с Pydantic Settings) |
| **AI-анализ** | OpenRouter API (httpx, async) |
| **Email** | `smtplib` (стандартная библиотека) |
| **Хранилище** | JSON-файлы на файловой системе |
| **Документация** | Swagger UI (/docs), ReDoc (/redoc) |
| **Менеджер зависимостей** | pip, `requirements.txt` |

**Почему эти технологии?**

- **FastAPI** — современный, производительный, асинхронный. Встроенная генерация OpenAPI-спецификации и Swagger.
- **Pydantic v2** — runtime-валидация на границе входа (аналог Zod в TypeScript). Автоматически генерирует схемы для OpenAPI.
- **Файловое JSON-хранилище** — вместо полноценной БД. Для портфолио не требуется СУБД — JSON-файлы проще в развёртывании и обслуживании.
- **httpx** — async HTTP-клиент (замена `requests` для неблокирующего I/O при общении с OpenRouter).

### Паттерны

Проект следует **многослойной (layered) архитектуре**, вдохновлённой чистой архитектурой:

```
HTTP Request
    ↓
Router (Controller) — маршрутизация, DI, HTTP-статусы
    ↓
Service Layer — бизнес-логика (AI + Email)
    ↓
Repository Layer — работа с данными (JSON)
    ↓
File System — хранение
```

Каждый слой отвечает только за свою зону ответственности:

- **Router** — HTTP, валидация, DI, ответы клиенту.
- **Service** — бизнес-логика, вызов внешних API.
- **Repository** — детали хранения (файловая система).

**Dependency Injection (DI):** Зависимости (`AIService`, `EmailService`, репозитории) передаются через `Depends()` FastAPI, что упрощает тестирование и замену компонентов.

## API

### POST /api/contact

Основной эндпоинт. Принимает форму обратной связи, анализирует её через AI, отправляет email и сохраняет статистику.

**Request:**

```json
{
  "name": "Иван Петров",
  "phone": "+7 999 123 45 67",
  "email": "ivan@example.com",
  "comment": "Отличное портфолио! Хочу обсудить проект."
}
```

**Success Response (201):**

```json
{
  "success": true,
  "message": "Thank you! Your message has been received.",
  "sentiment": "позитивный",
  "category": "сотрудничество"
}
```

**Error Responses:**

- **422 Validation Error** — поля не прошли валидацию:

  ```json
  {
    "detail": [
      {
        "type": "string_too_short",
        "loc": ["body", "name"],
        "msg": "String should have at least 2 characters",
        "input": "A"
      }
    ]
  }
  ```

- **429 Rate Limit** — превышен лимит запросов:

  ```json
  {
    "detail": "Слишком много запросов. Попробуйте через 1845 секунд."
  }
  ```

  Дополнительно возвращается HTTP-заголовок `Retry-After`.

**Валидация полей:**

| Поле | Тип | Ограничения |
|------|-----|-------------|
| `name` | string | 2–100 символов |
| `phone` | string | 7–20 символов |
| `email` | EmailStr | валидный email |
| `comment` | string | 10–2000 символов |

Все строки автоматически проходят trim (удаление лишних пробелов).

### GET /api/health

Проверка работоспособности сервиса. Подходит для Docker healthcheck и мониторинга.

**Response (200):**

```json
{
  "status": "ok",
  "service": "portfolio-api"
}
```

### GET /api/metrics

Агрегированная статистика обращений из файлового хранилища.

**Response (200):**

```json
{
  "total_contacts": 12,
  "sentiments": {
    "позитивный": 5,
    "негативный": 2,
    "нейтральный": 4,
    "неопределен": 1
  },
  "categories": {
    "сотрудничество": 3,
    "фидбек": 5,
    "вопрос": 2,
    "общее": 2
  }
}
```

## AI-интеграция

### Инструменты

- **Провайдер:** [OpenRouter](https://openrouter.ai/) — бесплатный роутер к множеству AI-моделей.
- **HTTP-клиент:** `httpx.AsyncClient` — асинхронные запросы (не блокируют event loop).
- **Модель по умолчанию:** `openrouter/free` (автоматически выбирает доступную бесплатную модель).

### Системный промпт

```text
Ты — ИИ-классификатор обращений. Твоя задача — проанализировать комментарий
пользователя на русском языке.

Верни СТРОГО один JSON-объект. Не пиши никакого текста до или после JSON.
Не используй разметку ```json.

Формат ответа строго такой (ключи на английском, значения на русском):
{"sentiment": "позитивный", "category": "вопрос"}

Допустимые значения для sentiment: позитивный, негативный, нейтральный.
Допустимые значения для category (примеры): оффер, сотрудничество, вопрос,
фидбек, спам, общее.

Примеры:
Пользователь: "Мне очень нравится ваш сайт!" ->
  {"sentiment": "позитивный", "category": "фидбек"}
Пользователь: "Как с вами связаться по работе?" ->
  {"sentiment": "нейтральный", "category": "сотрудничество"}
Пользователь: "Все работает ужасно, исправьте!" ->
  {"sentiment": "негативный", "category": "фидбек"}
```

Промпт содержит few-shot примеры для стабильности классификации. `response_format` принудительно установлен в `json_object`, а `temperature` — 0.1 для минимальной вариативности.

### Fallback-механизм

При любой ошибке (сетевой сбой, неверный JSON, пустой ответ, отсутствие API-ключа) сервис гарантированно возвращает:

```json
{"sentiment": "неопределен", "category": "общее"}
```

Это гарантирует, что форма обратной связи не сломается из-за проблем с AI.

## Хранение данных

Вместо базы данных используются JSON-файлы в директории `app/storage/`. Директория и файлы создаются автоматически при первом запуске.

### storage/metrics.json

Агрегированная статистика обращений.

```json
{
  "total_contacts": 12,
  "sentiments": {
    "позитивный": 5,
    "негативный": 2,
    "нейтральный": 4,
    "неопределен": 1
  },
  "categories": {
    "сотрудничество": 3,
    "фидбек": 5,
    "вопрос": 2,
    "общее": 2
  }
}
```

### storage/rate_limits.json

Временные метки запросов для rate limiting (скользящее окно).

```json
{
  "127.0.0.1": [
    "2025-04-10T12:00:00+00:00",
    "2025-04-10T12:05:00+00:00"
  ]

```

Лог-файл всех запросов в формате:

```

```

## Rate Limiting

Реализован паттерн **скользящего окна** на основе JSON-файла (`rate_limits.json`).

- **По умолчанию:** 5 запросов в час (3600 секунд) с одного IP.
- **Настройка:** через переменные `RATE_LIMIT_MAX_REQUESTS` и `RATE_LIMIT_WINDOW_SECONDS` в `.env`.
- **Механизм:** для каждого IP хранится список временных меток успешных запросов. При каждом новом запросе старые метки (выходящие за пределы окна) удаляются. Если количество меток в окне >= лимита, запрос отклоняется.
- **IP detection:** учитывает заголовок `X-Forwarded-For` для reverse-proxy.
- **Ответ при блокировке:** HTTP 429 + заголовок `Retry-After` с количеством секунд до следующего доступного слота.

## Что сделано с помощью AI

Весь проект написан с помощью AI-ассистентов (Claude, Copilot). Все промпты формулировались на русском языке.

### Что генерировалось автоматически

- Базовая структура FastAPI-приложения (main, router, схемы).
- Сервисы AI и email (начальная версия).
- Репозиторий для работы с JSON-файлами.
- Шаблон фронтенда (`index.html`).
- Данный README.md.

### Что было изменено/доработано вручную

1. **Переход с `requests` на `httpx`.** Изначальная генерация использовала синхронный `requests` внутри асинхронного контекста. Вручную заменён на `httpx.AsyncClient` для корректного async-кода.

2. **Настройка Dependency Injection.** Первоначально сервисы создавались внутри роутера. Вручную вынесены в `Depends()`-провайдеры с фабричными функциями.

3. **Локализация значений на русский язык.** AI-сгенерированный код содержал английские значения для sentiment (`"positive"`, `"negative"`). Вручную переведены в `Sentiment`-enum с русскими строками: `"позитивный"`, `"негативный"`, `"нейтральный"`, `"неопределен"`.

4. **Глобальный error handler.** Добавлен `@app.exception_handler(Exception)` для обработки необработанных исключений с возвратом 500 и русскоязычным сообщением об ошибке.

5. **Настройка rate limiting.** Добавлен слой безопасности: скользящее окно, `Retry-After` header, защита от злоупотреблений.

6. **Исправление ошибок сериализации.** AI-сгенерированные enum-поля не всегда корректно сериализовались в JSON. Вручную добавлена нормализация значений (`_normalize_sentiment`, `_normalize_category`).

### Вывод

AI значительно ускорил написание каркаса проекта, но понимание архитектуры, async-паттернов, безопасности и корректной работы с Pydantic v2 потребовало ручного вмешательства. Проект — пример эффективного симбиоза: AI генерирует черновик, разработчик доводит до production-качества.
